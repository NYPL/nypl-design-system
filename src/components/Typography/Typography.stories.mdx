# Typography

Consuming applications should utilize the DS standard text components as much as possible. In cases where that is not possible, the DS exposes CSS variables to assist with formatting text elements.

<Story name="Standard Text Components" />

## Standard Text Components

### Headings

Use the NYPL Design System [Heading](https://nypl.github.io/nypl-design-system/storybook-static/?path=/docs/heading--basic) component to render a standard HTML `<h>` tag. The `level` prop can be used to set the specific `<h>` tag that you require. Each heading level has a default style and size, but those styles can be overridden using the `displaySize` prop.

The display sizes are separate from semantic elements and may be used to override the default size and styling of the semantic HTML heading elements. For example, the `h1` element has the `Primary` style applied by default, but a new style can be applied through the `displaySize` prop. Any `displaySize` style can be applied to any `Heading` element.

`1rem` in the Design System is equal to 16px.

> <p className="heading--primary" style={{ color: "black" }}>
>   Primary
> </p>
>
> - --font-size-4 (2.25rem)
> - --font-weight-light

> <p className="heading--secondary" style={{ color: "black" }}>
>   Secondary
> </p>
>
> - --font-size-3 (1.75rem)
> - --font-weight-medium

> <p className="heading--tertiary" style={{ color: "black" }}>
>   Tertiary
> </p>
>
> - --font-size-2 (1.375rem)
> - --font-weight-medium

> <p className="heading--callout" style={{ color: "black" }}>
>   Callout
> </p>
>
> - --font-size-1 (1.25rem)
> - --font-weight-regular

### Body Text

Use the NYPL Design System `Text` component to render a standard HTML `<p>` tag. The Text component is generally used for body copy, captions and secondary captions. The `displaySize` prop can be used to render a specific style from the DS default text styles.

- [Typography: Body Styles](https://www.figma.com/file/qShodlfNCJHb8n03IFyApM/Main?node-id=10975%3A52)
- [Typography: Other Styles](https://www.figma.com/file/qShodlfNCJHb8n03IFyApM/Main?node-id=10975%3A100)

`1rem` in the Design System is equal to 16px.

> <p className="text" style={{ color: "black" }}>
>   Body
> </p>
>
> - --font-size-0 (1rem)

> <p className="text--caption" style={{ color: "black" }}>
>   Caption
> </p>
>
> - --font-size--1 (0.875rem)

> <p className="text--tag" style={{ color: "black" }}>
>   Tag
> </p>
>
> - --font-size--2 (0.75rem)

> <p className="text--mini" style={{ color: "black" }}>
>   Mini
> </p>
>
> - --font-size--3 (0.625rem)

<Story name="Font Family" />

## Font Family

With website accessibility in mind, font choices for the DS were made with an eye toward clarity and readability. Headings and body content use the following font family assignment: system-ui, sans-serif.

The following CSS variables are available through the DS.

```
--typeface-heading: system-ui, sans-serif;
--typeface-body: system-ui, sans-serif;
```

> **Recommendations**  
> When making font selections for project based components outside the DS, sans serif fonts display better on computers and mobile devices. Avoid decorative or overly stylized fonts, as they are often difficult to read even for users without visual impairments or reading Standard Text Components

<Story name="Font Colors" />

## Font Colors

All text within a project, unless specifically noted in mockups, should be black on a white background.

In cases where specific colors must be applied to text elements, the DS provides CSS variables for the full NYPL color palette. Refer to the [Colors](https://nypl.github.io/nypl-design-system/storybook-static/?path=/docs/style-guide-colors--colors-brand) section of the Style Guide for details.

<Story name="Font Weight" />

## Font Weight

The DS uses four font weights to render text. The default font-weight is 300.

<p
  style={{
    fontSize: "1rem",
    fontWeight: 300,
    fontFamily: "system-ui, sans-serif",
  }}
>
  Light / 300
</p>
<p
  style={{
    fontSize: "1rem",
    fontWeight: 400,
    fontFamily: "system-ui, sans-serif",
  }}
>
  Regular / 400
</p>
<p
  style={{
    fontSize: "1rem",
    fontWeight: 500,
    fontFamily: "system-ui, sans-serif",
  }}
>
  Medium / 500
</p>
<p
  style={{
    fontSize: "1rem",
    fontWeight: 700,
    fontFamily: "system-ui, sans-serif",
  }}
>
  Bold / 700
</p>

The following CSS variables are available through the DS.

```
--font-weight-light: 300; /* default */
--font-weight-regular: 400;
--font-weight-medium: 500;
--font-weight-bold: 700;
```

<Story name="Font Sizing" />

## Font Sizing

Font sizing is based on a root font size of 16px (1rem = 16px).

The following CSS variables are available through the DS.

```
--font-size--3: 0.625rem;
--font-size--2: 0.75rem;
--font-size--1: 0.875rem;
--font-size-0: 1rem; /* default */
--font-size-1: 1.125rem;
--font-size-2: 1.375rem;
--font-size-3: 1.75rem;
--font-size-4: 2.25rem;
```

<Story name="Text Case" />

## Text Case

In general, the New York Public Library uses Title Case for titles. This means only using capital letters for the principal words. Articles, conjunctions, and prepositions do not get capital letters unless they start the title. For example:

- Snow White and the Seven Dwarfs
- The Last of the Mohicans
- Spooky Books for Kids: The Ultimate Creepy Guide
- The Future Teen Stars of America Live on TikTok
- How Jacqueline Woodson Captures Every Side of Brooklyn

<Story name="Character Count" />

## Character Count

- **Titles**: 80 characters max, 60 characters recommended
- **Summaries**: 140 characters max<br />Note: Online Exhibitions, with their longer, legacy summaries are an exception to this
- **Page Intros**: No max character count, 200 characters recommended
