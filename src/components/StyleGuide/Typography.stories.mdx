## General Information

Consuming applications should utilize the Reservoir Design System (DS) standard
text components as much as possible. In cases where that is not possible, the
DS exposes CSS variables to assist with formatting text elements.

The sizing of all text elements in the Design System is based on `1rem` being
equal to `16px`.

## Heading Component

Use the DS [Heading](/docs/components-typography-styles-heading--heading-props)
component to render a standard HTML `<h>` tag. The `level` prop can be used to
set the specific `<h>` tag that you require.

<Canvas>
  <DSProvider>
    <Heading level="one">Heading Level 1</Heading>
    <Heading level="two">Heading Level 2</Heading>
    <Heading level="three">Heading Level 3</Heading>
    <Heading level="four">Heading Level 4</Heading>
  </DSProvider>
</Canvas>

#### Heading Display Sizes

<DSProvider>

Each heading level has a default style and size, but those styles can be overridden
using the `size` prop. The display sizes are separate from semantic elements and
may be used to override the default size and styling of the semantic HTML heading
elements. For example, the `h1` element has the `Primary` style applied by default,
but a new style can be applied through the `size` prop. Any `size` style can be
applied to any `Heading` element.

## Text Component

Use the DS [Text](/docs/components-typography-styles-text--text)
component to render a standard HTML `<p>` tag. The Text component is generally
used for body copy, captions and secondary captions.

<Canvas>
  <DSProvider>
    <Text>
      Curabitur blandit tempus porttitor. Cras mattis consectetur purus sit amet
      fermentum. Nullam id dolor id nibh ultricies vehicula ut id elit. Maecenas
      faucibus mollis interdum. Morbi leo risus, porta ac consectetur ac,
      vestibulum at eros.
    </Text>
  </DSProvider>
</Canvas>

## Font Family

With website accessibility in mind, font choices for the DS were made with an
eye toward clarity and readability. Headings and body content use the
following font family assignment:

<Canvas>
  <DSProvider>
    <Text>system-ui, sans-serif</Text>
  </DSProvider>
</Canvas>

The following CSS variables are available through the DS.

```
--nypl-fonts-heading: system-ui, sans-serif;
--nypl-fonts-body: system-ui, sans-serif;
```

> **RECOMMENDATIONS**  
> When making font selections for project based components outside the DS,
> sans serif fonts display better on computers and mobile devices. Avoid
> decorative or overly stylized fonts, as they are often difficult to read even
> for users without visual impairments or reading Standard Text Default

## Font Color

All text within a project, unless specifically noted in mockups, should be
black on a white background.

<Canvas>
  <DSProvider>
    <Heading level="three">Default Heading Color</Heading>
    <Text>
      This paragraph shows the default body text color. Curabitur blandit tempus
      porttitor. Praesent commodo cursus magna, vel scelerisque nisl consectetur
      et. Integer posuere erat a ante venenatis dapibus posuere velit aliquet.
      Duis mollis, est non commodo luctus, nisi erat porttitor ligula, eget
      lacinia odio sem nec elit.
    </Text>
  </DSProvider>
</Canvas>

In cases where specific colors must be applied to text elements, the DS
provides CSS variables for the full NYPL color palette. Refer to the
[Colors](https://nypl.github.io/nypl-design-system/reservoir/v1/?path=/docs/style-guide-colors--page)
section of the Style Guide for details.

## Font Weight

The DS uses four font weights to render text. The default font-weight is 300.

<Canvas>
  <>
    <p
      style={{
        fontSize: "var(--nypl-fontSizes-0)",
        fontWeight: "var(--nypl-fontWeights-light)",
        fontFamily: "var(--nypl-fonts-body)",
      }}
    >
      Light / 300 (default)
    </p>
    <p
      style={{
        fontSize: "var(--nypl-fontSizes-0)",
        fontWeight: "var(--nypl-fontWeights-regular)",
        fontFamily: "var(--nypl-fonts-body)",
      }}
    >
      Regular / 400
    </p>
    <p
      style={{
        fontSize: "var(--nypl-fontSizes-0)",
        fontWeight: "var(--nypl-fontWeights-medium)",
        fontFamily: "var(--nypl-fonts-body)",
      }}
    >
      Medium / 500
    </p>
    <p
      style={{
        fontSize: "var(--nypl-fontSizes-0)",
        fontWeight: "var(--nypl-fontWeights-bold)",
        fontFamily: "var(--nypl-fonts-body)",
      }}
    >
      Bold / 700
    </p>
  </>
</Canvas>

The following design tokens are available through the DS.

|             | Value | JS Theme Object         | CSS Variable                                   |
| ----------- | ----- | ----------------------- | ---------------------------------------------- |
| **Light**   | `300` | `fontWeight: "light"`   | `font-weight: var(--nypl-fontWeights-light)`   |
| **Regular** | `400` | `fontWeight: "regular"` | `font-weight: var(--nypl-fontWeights-regular)` |
| **Medium**  | `500` | `fontWeight: "medium"`  | `font-weight: var(--nypl-fontWeights-medium)`  |
| **Bold**    | `700` | `fontWeight: "bold"`    | `font-weight: var(--nypl-fontWeights-bold)`    |

## Text Case

In general, the New York Public Library uses Title Case for titles. This means
only using capital letters for the principal words. Articles, conjunctions, and
prepositions do not get capital letters unless they start the title. For example:

<Canvas>
  <DSProvider>
    <List type="ul">
      <li>Snow White and the Seven Dwarfs</li>
      <li>The Last of the Mohicans</li>
      <li>Spooky Books for Kids: The Ultimate Creepy Guide</li>
      <li>The Future Teen Stars of America Live on TikTok</li>
      <li>How Jacqueline Woodson Captures Every Side of Brooklyn</li>
    </List>
  </DSProvider>
</Canvas>

## Character Count

### Titles

80 characters max, 60 characters recommended

<Canvas>
  <DSProvider>
    <Heading level="three">
      This is a Long Title to Show What 58 Characters Looks Like
    </Heading>
  </DSProvider>
</Canvas>

### Summaries

140 characters max<br />
Note: Online Exhibitions, with their longer, legacy summaries are an exception
to this.

<Canvas>
  <DSProvider>
    <Text>
      This block of text shows what 140 chacters and spaces looks like. Integer
      posuere erat a ante venenatis dapibus posueret aliquet nullas non.
    </Text>
  </DSProvider>
</Canvas>

### Page Intros

No max character count, 200 characters recommended
