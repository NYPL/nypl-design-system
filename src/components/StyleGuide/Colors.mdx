## Overview

The Reservoir Design System (DS) provides the NYPL branded and approved
color palette. The NYPL color palette is used to style the DS components and
is also available for consuming applications in the form of javascript and
CSS design tokens. Please refer to the [Design Tokens](../?path=/docs/development-guide-design-tokens--docs)
page for information about consuming and using all style tokens.

Whenever using these colors, designer must ensure that the foreground and
background colors pass, at a minimum, the `WCAG 2.1 AA` accessibility guidelines for
color contrast. For reference, to meet the `WCAG 2.1 AA` standard, the contrast ratio
must be at least 4.5:1. To meet the `WCAG 2.1 AAA` standard, the contrast ratio must
be at least 7:1.

Resources:

- [W3C WCAG 2.1 Contrast Minimum](https://www.w3.org/TR/WCAG21/#contrast-minimum)

NOTE: There are a handful of colors that are semi-transpararent. Colors that use
transparency are tough to talk about when it comes to color combinations and
color contrast. That being the case, we have used question marks for the
contrast values and WCAG status.

## UI Colors

UI color are not connected to the NYPL brand and are intended to be generic.

### Semantic Colors

Semantic colors are named to match their function. They are generally used in
common UI and typography patterns, and to communicate component and/or
application state.

<Box
  border="1px"
  borderColor="ui.border.default"
  borderRadius="5px"
  mb="m"
  p="s"
>
  <SimpleGrid columns={1} gap="s">
    {getColorCards("ui")}
  </SimpleGrid>
</Box>

### Grayscale Colors

Grayscale colors are used as a base for the semantic colors. Outside of
`ui.black` and `ui.white`, these colors should not be used directly in designs
and coding.

  border="1px"
  borderColor="ui.border.default"

## Dark Mode UI Colors

The following "dark mode" color fills are used for body copy and to communicate
component and/or application state in dark mode.

These colors are not connected to the NYPL brand, but are intended to be generic.
