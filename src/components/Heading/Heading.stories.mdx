# Heading

| Component Version | DS Version |
| ----------------- | ---------- |
| Added             | `0.0.4`    |
| Latest            | `0.25.9`   |

<Description of={Heading} />

A `Heading` component will render a standard HTML `<h>` tag (1-6). The heading's
text can be passed in through a `text` prop or as a child. Default styles for
semantic elements can be overwritten using the `displaySize` prop.

<Canvas withToolbar>

      text: "Default Heading",
      url: undefined,
      urlClass: undefined,
    }}
  >
    {(args) => (
      <Heading
        {...args}
        displaySize={sizesEnumValues.getValue(args.displaySize)}
        level={levelsEnumValues.getValue(args.level)}

</Canvas>

## Headings with Links

When the `url` prop is passed to `Heading`, a child `<a>` element is created and the heading text becomes an active link.

<Canvas>
  <Story name="Headings with Links">
    <Heading
      id="heading-with-link-url"
      level={HeadingLevels.Two}
      text="Heading with URL Props"
      url="http://apple.com"
      urlClass="special-link"
    />
    <Heading id="heading-with-link-child" level={HeadingLevels.Two}>
      <>
        Heading with a Word <a href="#hello">Link</a>
      </>
    </Heading>
  </Story>
</Canvas>
