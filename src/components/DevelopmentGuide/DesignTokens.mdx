## What are Design Tokens?

_"With design tokens, you can capture low-level values and then use them to
create the styles for your product or app. You can maintain a scalable and
consistent visual system for UI development."_ - Jina Bolton, Lead Designer for
the Salesforce Design System

Design tokens are a collection of style values &ndash; spacing, colors,
typography, breakpoints, and more &ndash; used throughout a design system. They
are a single source of truth for design styles, bringing consistency and
cohesion to components within a design system, and they allow teams to more
quickly and easily build various applications with the same look and feel.

## How to Use the Reservoir Design Tokens

The NYPL design tokens are built into the Reservoir Design System (DS) and are
offered in two formats for consumption: Javascript theme objects (recommended)
and CSS variables. To access either of these formats, the DS `v0.25.9` or higher
needs to be properly installed in a consuming React application.

## Using Reservoir Design Tokens in Javascript

It is recommended that consuming apps use Javascript (rather than CSS) to apply
styles to UI elements whenever possible.

To use the DS <Link href="#reservoir-design-token-categories"
target="_self">Javascript design tokens</Link>, the `DSProvider` wrapper
component must first be imported and implemented properly in a consuming app.
Please refer to the `DSProvider` section of the [DS Chakra
UI](../?path=/docs/chakra-ui--docs#dsprovider) page for instructions on how to
do this.

#### Design Tokens for DS and Chakra Components

**IMPORTANT:** The general rule of thumb is that the styles and functionality of
the DS components should not be altered. The components should be used with the
styles and variants as they are depicted in Storybook. There may be times,
however, when a component does require a modification. If a DS component does
need a style change in a consuming app, designers/developers should first
contact the Reservoir team about the change before it is implemented.

The Javascript design tokens can be used to populate the values for style props
added directly to the components. The shorthard nature of the Javascript tokens
make it easy to quickly and logically add styles to a DS component.

For example, if a `Heading` component needs to sit within a wrapper element and
render its text using Reservoir's `brand primary` color and use a `light`
font weight, the following can be done:

<Source
  code={`
<Box
  backgroundColor="ui.bg.default"
  border="1px solid"
  borderColor="ui.border.default"
  padding="inset.default"
>
  <Heading color="brand.primary" fontWeight="light">
    Get a Digital Library Card Today in a Few Easy Steps
  </Heading>
</Box>
`}
  language="jsx"
/>

#### Design Tokens for Custom Components and Native HTML Elements

**IMPORTANT:** Javascript design tokens **cannot be used** with custom
components and native HTML elements as they are not compatible with the way
styles are passed into these elements. Instead, CSS design tokens must be used.

To pass style attributes to custom components and native HTML elements, the
React `style` prop must be used rather than the individual Javascript style
props. The `style` prop can be populated in a number of ways.

<Source
  code={`
// Style values assigned to variables
const bg = "var(--nypl-colors-ui-bg-default)";
const color = "var(--nypl-colors-ui-typography-heading)";
const fontWeight = "var(--nypl-fontWeights-medium)";
const padding = "var(--nypl-space-inset-default)";
// ...
<div
  style={{
    backgroundColor: bg,
    color: color,
    fontWeight: fontWeight,
    padding: padding,
  }}
/>
`}
  language="jsx"
/>

<Source
  code={`
// Style values assigned to a data object
const stylesDirectlyAssigned = {
  backgroundColor: "var(--nypl-colors-ui-bg-default)",
  color: "var(--nypl-colors-ui-typography-heading)",
  fontWeight: "var(--nypl-fontWeights-medium)",
  padding: "var(--nypl-space-inset-default)",
};
// ...
<div style={stylesDirectlyAssigned} />
`}
  language="jsx"
/>

<Source
  code={`
// Style values assigned directly
<div
  style={{
    backgroundColor: "var(--nypl-colors-ui-bg-default)",
    color: "var(--nypl-colors-ui-typography-heading)",
    fontWeight: "var(--nypl-fontWeights-medium)",
    padding: "var(--nypl-space-inset-default)",
  }}
/>
`}
  language="jsx"
/>

Below is an example using a custom component. Please note that consuming apps
are responsible for handling how styles are propagated and applied within custom
components.

<Source
  code={`
// Styles for a custom component
const stylesForCustomComponent = {
  backgroundColor: "var(--nypl-colors-ui-bg-default)",
  color: "var(--nypl-colors-ui-typography-heading)",
  fontWeight: "var(--nypl-fontWeights-medium)",
  padding: "var(--nypl-space-inset-default)",
};
// ...
<CustomComponent style={stylesForCustomComponent} />
`}
  language="jsx"
/>

## Using Reservoir Design Tokens in CSS

If your app uses CSS or SCSS to handle styles, the DS creates standard CSS
custom properties (e.g. variables) for consumption in your stylesheets. If the
DS is properly installed, the CSS custom properties will be available for you to
use. Nothing more needs to be done to your app.

To assist with CSS scope isolation, all of the CSS custom properties created by
the DS are prefixed with "nypl." Please see the examples below.

<Source
  code={`
--nypl-colors-brand-primary
--nypl-colors-brand-secondary
--nypl-fontSizes-sm
--nypl-fontSizes-md
--nypl-fontSizes-lg
--nypl-fontSizes-xl
--nypl-space-xxs
--nypl-space-xs
--nypl-space-s
--nypl-space-m
--nypl-space-l
`}
  language="css"
/>

The CSS custom properties are defined on the :root pseudo-class and should be
implemented using standard CSS custom properties conventions.

<Source
  code={`
/* Property Declaration in the DS */
:root {
  --nypl-colors-brand-primary: #c60917;
}
/* Property Usage in Your Application */
.custom-button {
  background: var(--nypl-colors-brand-primary);
}
`}
  language="css"
/>

- `fill`
- `stroke`

### Typography

The `typography` design tokens include options for the following styles:

- `fontFamily` (js) and `font-family` (css)
- `fontSize` (js) and `font-size` (css)

For more information about NYPL typography, see the [Typography Style
Guide](../?path=/docs/style-guide-typography--docs) page.
